\begin{abstract}
We summarise the experimental protocol, fairness assessment, compute governance, and reproducibility
controls for the IEEE FX Forecasting benchmark. The accompanying assets, including source data,
analysis notebooks, and automation scripts, are tracked under version control in this repository.
\end{abstract}

\section{Protocol Overview}
Our benchmark follows a mixed-frequency protocol combining daily macroeconomic signals with
high-frequency order book snapshots. Data preprocessing adheres to the standardisation pipeline
implemented in \texttt{src/data/pipelines.py}, ensuring consistent feature scaling, calendar alignment,
and leakage-free train/validation/test splits. Hyperparameters for the mixture-of-experts architecture
are specified in \texttt{configs/models/moe.yaml}, and optimisation runs are orchestrated with the
Makefile targets \texttt{make train} and \texttt{make evaluate}. Table~\ref{tab:fairness} enumerates
the fairness metrics generated by the automated evaluation workflow.

\section{Fairness Measures}
We evaluate parity across currency regions using accuracy, true positive rate (TPR), and false positive
rate (FPR). The metrics in Table~\ref{tab:fairness} are derived from the CSV artifact located at
\texttt{docs/dmlr/tables/fairness\_metrics.csv}, which is produced by the benchmarking pipeline and
versioned alongside this manuscript. The robust configuration narrows the gap between USD and EUR
markets, reducing FPR disparities while improving TPR balance. These findings motivate the inclusion of
regional weighting during mixture component selection.

\section{Compute Governance}
Figure~\ref{fig:compute-governance} visualises the resource allocation budget for training, evaluation,
and post-deployment monitoring. The energy and GPU-hour traces originate from the governance log
exporter (\texttt{scripts/emit\_governance\_budget.py}), and the summarised dataset is versioned at
\texttt{docs/dmlr/figures/compute\_governance\_data.csv}. We integrate resource caps via the
\texttt{configs/governance.yaml} policy, enabling automated enforcement of carbon-aware scheduling and
cost ceilings. Weekly audits verify alignment with organisational sustainability commitments.

\section{Reproducibility and Governance}
Reproducibility is supported through deterministic data splits, containerised execution (see
\texttt{Dockerfile}), and pinned dependencies defined in \texttt{environment.yml}. The manuscript is
linked to repository commit \texttt{\input{|git rev-parse HEAD}} and DOI
\href{https://doi.org/10.5281/zenodo.pending-ieee-fx}{10.5281/zenodo.pending-ieee-fx}. We distribute a
submission checklist (Appendix~\ref{sec:checklist}) capturing protocol compliance, and provide
step-by-step submission instructions under version control for audit trails.
